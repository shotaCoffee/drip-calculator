# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev      # Start dev server (webpack mode, http://localhost:3000)
npm run build    # Production build (webpack mode)
npm run start    # Start production server after build
npm run lint     # Run ESLint

node scripts/generate-assets.mjs  # Regenerate PWA icons and OGP image
```

> `--webpack` is forced in dev/build scripts because `@ducanh2912/next-pwa` requires webpack and Next.js 16 defaults to Turbopack.

## Architecture

This is a **fully static Next.js 16 App Router** app (no API routes, no server components with data fetching). All pages are pre-rendered as static HTML at build time.

### Key files

- `components/DripCalculator.tsx` — the entire app UI and logic as a single `'use client'` component. Contains the `RECIPES` constant (all ratio/temp/grind data), `ResultRow` and `GrindRow` sub-components, and `useState` for `mode`, `strength`, and `grams`.
- `app/page.tsx` — wraps `DripCalculator` and injects a JSON-LD `WebApplication` structured data script.
- `app/layout.tsx` — sets OGP/Twitter metadata, loads fonts via `next/font/google` (`DM_Sans` → `--font-num`, `Noto_Sans_JP` → `--font-body`), and sets `metadataBase`.
- `app/globals.css` — all styles using CSS custom properties. No CSS Modules or Tailwind.
- `app/manifest.ts`, `app/robots.ts`, `app/sitemap.ts` — Next.js route handlers for PWA manifest, robots.txt, and sitemap.xml.
- `scripts/generate-assets.mjs` — Node.js script using `sharp` to generate `public/icon-192.png`, `public/icon-512.png`, and `public/og-image.png` from inline SVG.

### Styling convention

Global CSS only (`app/globals.css`). CSS variables are defined in `:root`. Mode-specific styles use class selectors like `.mode-hot` and `.mode-ice` on the card wrapper, e.g. `.mode-hot .result-value { color: var(--accent); }`.

### Recipe data

All drip recipes live in the `RECIPES` constant in `DripCalculator.tsx`. To change ratios, temperatures, or grind recommendations, edit that object directly.

### PWA

Service worker (`public/sw.js`) is auto-generated by `@ducanh2912/next-pwa` on each build. Do not edit it manually. PWA is disabled in development (`disable: process.env.NODE_ENV === 'development'` in `next.config.ts`).

### Asset generation

If icons or OGP image need to be updated, edit the SVG strings in `scripts/generate-assets.mjs` and re-run `node scripts/generate-assets.mjs`.
